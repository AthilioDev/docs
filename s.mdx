# Bloqueio por IP

O AthilioFy utiliza um sistema de autenticação com validação por IP para aumentar a segurança da licença e impedir ativações não autorizadas.

Esse processo ocorre automaticamente durante a inicialização do recurso.

---

# Como Funciona

Ao iniciar o servidor, o sistema realiza automaticamente:

- Verificação da licença
- Identificação do IP do VPS
- Validação da autenticação
- Registro da tentativa de conexão

Caso o IP esteja autorizado, a licença será ativada normalmente:

```txt
[Athilio Spotify] ✔ Licença validada com sucesso!
```

---

# IP Não Autorizado

Quando o IP do servidor não corresponde ao autorizado no painel, o sistema retornará:

```txt
[Athilio Spotify] ✖ IP não autorizado
```

## Possíveis Motivos

- Migração de VPS
- Alteração de IP da hospedagem
- Proxy ou firewall interferindo
- Configuração incorreta do servidor
- Utilização da licença em outro ambiente

<Warning>
  Nunca compartilhe sua licença publicamente ou utilize a mesma chave em múltiplos servidores sem autorização.
</Warning>

---

# Redefinição de IP

Caso o IP do servidor seja alterado, realize uma redefinição diretamente no dashboard da Athilio Software para atualizar o novo endereço autorizado.

## Quando redefinir

- Troca de hospedagem
- Mudança de VPS
- Reinstalação do servidor
- Alteração de IP público
- Mudança de infraestrutura

<Tip>
  Sempre confirme o IP correto do VPS antes de realizar a redefinição no painel.
</Tip>

---

# Histórico de Validações

Todas as tentativas de autenticação ficam registradas no painel:

```txt
18/05/26 21:57:04 | 0.0.0.0 | 30120 | Sucesso | OK
17/05/26 23:03:20 | 0.0.0.0 | 30120 | Falhou  | Licença não encontrada
```

Cada tentativa armazena:

- Data e horário
- Endereço IP
- Porta utilizada
- Status da autenticação
- Mensagem retornada pelo sistema

---

# Segurança

O sistema de bloqueio por IP garante:

- Integridade da licença
- Controle de ativações
- Segurança operacional
- Redução de compartilhamentos indevidos
- Monitoramento de autenticações

---

# Recomendação

Sempre mantenha:

- O IP atualizado no painel
- A licença configurada corretamente
- O nome original do recurso
- O servidor conectado à internet
