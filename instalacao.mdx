O **AthilioFy** foi desenvolvido para oferecer instalação rápida, estável e compatível com qualquer estrutura moderna do FiveM.

# Processo de Instalação

## 1. Extraia o recurso

Envie a pasta:

```txt
athilio_spotify
```

Para:

```txt
resources/[scripts]
```

Estrutura esperada:

```txt
resources/
└── [scripts]/
    └── athilio_spotify/
```

---

## 2. Configure a licença

Abra o arquivo:

```txt
athilio_spotify/license.json
```

Configure sua licença:

```json
{
  "license": "SUA_LICENSE_AQUI"
}
```

---

> Importante
>
> Nunca compartilhe sua licença publicamente.
>
> O sistema realiza validação automática durante a inicialização do recurso.

---

## 3. Adicione no server.cfg

Adicione:

```cfg
ensure athilio_spotify
```

Recomendado iniciar após sua framework principal.

Exemplo:

```cfg
ensure vrp
ensure oxmysql
ensure athilio_spotify
```

Porque ordem de inicialização ainda consegue quebrar servidor inteiro por causa de dependência carregada errado. O ser humano inventou inteligência artificial antes de aprender a organizar `server.cfg`.

---

## 4. Reinicie o servidor

Após iniciar o recurso, o sistema executará automaticamente:

* Verificação da licença
* Inicialização da interface
* Carregamento da NUI
* Sincronização interna
* Proteções de autenticação
* Verificações de integridade

# Problemas Comuns

## Licença inválida

Verifique:

* Licença configurada corretamente
* IP autorizado no painel
* Nome correto da pasta
* VPS conectada à internet
* Firewall liberando conexões externas

---

## Recurso não inicia

Confirme se existe:

```cfg
ensure athilio_spotify
```

No arquivo:

```txt
server.cfg
```

Também valide:

* Nome da pasta exatamente igual
* Artefact atualizado
* Dependências iniciadas corretamente

---

## Interface não abre

Verifique:

* Pasta `html/` presente
* Arquivos NUI íntegros
* Cache do FiveM limpo
* Erros no console F8

# Recomendação

> Utilize artefacts atualizados do FiveM para garantir melhor estabilidade, compatibilidade e segurança operacional.

Também é recomendado:

* Utilizar VPS Linux atualizada
* Manter `oxmysql` em versões recentes
* Evitar alterações estruturais no recurso
* Não renomear arquivos internos

Porque sempre aparece alguém que muda nome de arquivo crítico e depois culpa o script. A tradição da computação moderna.
