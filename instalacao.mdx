# Processo de Instalação

## 1. Extraia o recurso

Envie a pasta `athilio_spotify` para o diretório de resources do servidor:

```txt
resources/
└── [scripts]/
    └── athilio_spotify/
```

---

## 2. Configure a licença

Abra o arquivo:

```txt
athilio_spotify/license.json
```

Configure sua licença:

```json
{
  "license": "SUA_LICENSE_AQUI"
}
```

<Warning>
  Nunca compartilhe sua licença publicamente. O sistema realiza validação automática durante a inicialização do recurso.
</Warning>

---

## 3. Adicione no server.cfg

Adicione ao final do seu `server.cfg`, preferencialmente após sua framework principal:

```cfg
ensure vrp
ensure oxmysql
ensure athilio_spotify
```

<Tip>
  A ordem de inicialização importa. Certifique-se de que o `athilio_spotify` é carregado após as dependências do servidor.
</Tip>

---

## 4. Reinicie o servidor

Após iniciar o recurso, o sistema executará automaticamente:

- Verificação da licença
- Inicialização da interface
- Carregamento da NUI
- Sincronização interna
- Proteções de autenticação
- Verificações de integridade

# Problemas Comuns

## Licença inválida

Verifique:

- Licença configurada corretamente no `license.json`
- IP autorizado no painel
- Nome correto da pasta
- VPS conectada à internet
- Firewall liberando conexões externas

## Recurso não inicia

Confirme se existe `ensure athilio_spotify` no `server.cfg` e valide:

- Nome da pasta exatamente igual ao original
- Artefact atualizado
- Dependências iniciadas corretamente

## Interface não abre

Verifique:

- Pasta `html/` presente e íntegra
- Cache do FiveM limpo
- Erros no console F8
